Docker大家应该都不会陌生，它是一个进程级的轻量虚拟机，依托容器技术，是现在跨平台部署特别有用的一个工具。

选择Docker是因为目标机器是离线的，不能联网，在不联网的情况下很难处理python的各种依赖包，因此选择Docker

4. 说明与体会

docker save 和docker load是一起的，docker export和docker import的区别？

核心记住两点：

  1. export / import 操作对象是“容器”（container），导出的是裸文件系统 tar——没有镜像层、没有历史、没有元数据。
  2. save / load 操作对象是“镜像”（image），导出的是完整镜像 tar——包含所有层、历史、repo:tag、构建缓存等。

绝对不能混用。

4.1 常见误区

  1. 用 export 想保存“镜像”——结果丢失所有元数据，启动容器会报错“/bin/sh not found”之类。
  2. 用 save 去保存容器——命令会直接拒绝，save 只能接镜像名。
  3. import 后忘了重新写 Dockerfile 或 docker run 参数——导致“镜像能跑起来却端口不对、命令不对”。
  4. 把 export/import 当成“精简镜像”手段——其实只是压扁层，并不会比 docker build --squash 更优，且失去缓存。
